## 3. End-to-End Training Data and Tensor Pipeline

### 3.1 Runtime flow
```mermaid
flowchart LR
    A[RawTrace from adapter] --> B[PrefixPolicy]
    B --> C[PrefixSlice]
    C --> D[FeatureEncoder]
    D --> E[DynamicGraphBuilder or BaselineGraphBuilder]
    E --> F[GraphTensorContract]
    F --> G[PyG Data]
    G --> H[PyG DataLoader batch]
    H --> I[Model forward]
    I --> J[logits]
    J --> K[cross entropy with y]
```

### 3.2 Tensor assembly ownership
1. `PrefixPolicy` defines target event (`target_event`) for each prefix.
2. `FeatureEncoder` converts event features to split node tensors (`x_cat`, `x_num`).
3. `BaselineGraphBuilder` builds observed graph tensors (`edge_index`, `edge_type`, `y`, `batch`).
4. `DynamicGraphBuilder` optionally injects structural tensors (`allowed_target_mask`, `structural_edge_index`, `structural_edge_weight`, `struct_x`).
5. `ModelTrainer` maps contract to PyG `Data` and back to forward contract.

## 4. Target Generation Rule (`y`)

### 4.1 Canonical rule
`y` is always the next event from `PrefixSlice.target_event`.

`PrefixSlice` is generated by all-prefix slicing:
1. observed prefix: `events[0:k]`,
2. target event: `events[k]`.

### 4.2 XES lifecycle filtering impact
For XES path, adapter-level lifecycle filtering is applied before prefix generation:
1. start-like transitions are used only for `start_ts` pairing,
2. by default only complete-like transitions become training events,
3. therefore, by default `y` is the next complete-like event.

```mermaid
sequenceDiagram
    participant X as XESAdapter
    participant P as PrefixPolicy
    participant B as GraphBuilder
    participant T as Trainer

    X->>X: filter lifecycle transitions
    X->>P: RawTrace(events after filtering)
    P->>P: generate PrefixSlice(target_event=next event)
    P->>B: PrefixSlice
    B->>T: GraphTensorContract(y from target_event)
```

## 5. GraphTensorContract Semantics

### 5.1 Baseline-required tensors
1. `x_cat: LongTensor [N, C_cat]`
2. `x_num: FloatTensor [N, C_num]`
3. `edge_index: LongTensor [2, E_obs]`
4. `edge_type: LongTensor [E_obs]`
5. `y: LongTensor [1]` per sample, `[B]` in batch
6. `batch: LongTensor [N]`

### 5.2 Optional structural tensors
1. `allowed_target_mask: BoolTensor [C]` per sample, `[B, C]` in batch
2. `structural_edge_index: LongTensor [2, E_struct]`
3. `structural_edge_weight: FloatTensor [E_struct]`
4. `struct_x: FloatTensor [C, F_struct]` (stats-backed structural node features)

### 5.3 Snapshot metadata tensors
1. `stats_snapshot_version_seq: int | None`
2. `stats_snapshot_as_of_epoch: float | None`
3. `stats_allowed: bool | None`
4. batch diagnostic variants for snapshot ids/timestamps.

## 6. Model Architecture Behavior

### 6.1 BaselineGATv2
Observed graph only:
1. embeddings for categorical node features,
2. two GATv2 layers,
3. global pooling,
4. linear classifier.

```mermaid
flowchart TD
    XCAT[x_cat] --> EMB[Embedding lookup]
    XNUM[x_num] --> CAT[Concat]
    EMB --> CAT
    CAT --> G1[GATv2Conv layer 1]
    G1 --> G2[GATv2Conv layer 2]
    G2 --> POOL[global_mean or last_node pooling]
    POOL --> CLS[Linear classifier]
    CLS --> LOGITS[logits]
```

### 6.2 BaselineGCN
Observed graph only:
1. embeddings for categorical node features,
2. three GCN layers with residual where shape-compatible,
3. global pooling,
4. linear classifier.

```mermaid
flowchart TD
    XCAT[x_cat] --> EMB[Embedding lookup]
    XNUM[x_num] --> CAT[Concat]
    EMB --> CAT
    CAT --> G1[GCNConv layer 1]
    G1 --> G2[GCNConv layer 2]
    G2 --> G3[GCNConv layer 3]
    G3 --> POOL[global_mean or last_node pooling]
    POOL --> CLS[Linear classifier]
    CLS --> LOGITS[logits]
```

### 6.3 EOPKGGATv2
Dual-encoder path:
1. observed branch: local GATv2 encoder,
2. structural branch: `struct_x` or structural embeddings + structural GNN,
3. cross-attention from observed context to structural nodes,
4. fused classifier head.

Fallback:
1. if structural edges are missing/empty, run baseline-like observed path only.

```mermaid
flowchart LR
    subgraph OBS[Observed branch]
        O1[x_cat and x_num] --> O2[Local GATv2 encoder]
        O2 --> O3[Pool to H_obs]
    end

    subgraph STR[Structural branch]
        S0{struct_x present}
        S1[struct_x]
        S2[struct node embedding]
        S3[Structural GNN]
        S0 -->|yes| S1
        S0 -->|no| S2
        S1 --> S3
        S2 --> S3
    end

    O3 --> A1[Cross-attention query]
    S3 --> A2[Cross-attention key and value]
    A1 --> A3[Cross-attention output]
    A2 --> A3
    A3 --> F1[Fusion]
    O3 --> F1
    F1 --> C1[Classifier]
    C1 --> LOGITS[logits]
```

### 6.4 EOPKGGCN
Compatibility EOPKG variant:
1. observed branch with GCN encoder,
2. structural summary context from structural tensors,
3. fusion MLP + classifier,
4. baseline fallback when structural tensors are missing.

```mermaid
flowchart LR
    O1[x_cat and x_num] --> O2[Local GCN encoder]
    O2 --> O3[Pool to H_obs]
    S1[Structural tensor summary] --> S2[Struct projector]
    O3 --> F1[Fusion MLP]
    S2 --> F1
    F1 --> C1[Classifier]
    C1 --> LOGITS[logits]
```

## 7. Evaluation Semantics
1. Loss is computed from `logits` and `y` using cross-entropy.
2. `allowed_target_mask` is not consumed by model forward.
3. `allowed_target_mask` is consumed by evaluator to compute OOS metrics when present.

```mermaid
flowchart LR
    CONTRACT[GraphTensorContract batch] --> MODEL[Model forward]
    MODEL --> LOGITS[logits]
    CONTRACT --> Y[y]
    LOGITS --> LOSS[cross entropy]
    Y --> LOSS
    CONTRACT --> MASK[allowed_target_mask optional]
    LOGITS --> OOS[OOS computation in evaluator]
    MASK --> OOS
```

## 8. Compatibility Rules
1. Baseline path must remain valid when all structural tensors are absent.
2. EOPKG path must degrade safely to observed-only forward when structural tensors are absent.
3. Optional tensor fields must never break MVP1-compatible execution.
